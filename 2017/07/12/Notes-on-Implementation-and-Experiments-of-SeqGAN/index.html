The basic structure of SeqGAN has been introduced in the previous article [Link]. The code of SeqGAN is available in Github, which includes the implementation and the synthetic data experiment.

In this article, there are two sections to be as a supplement for SeqGAN. In section of Implementation, crucial classes and functions are listed. And in the section of Synthetic Data Experiment, the flow of the experiment is introduced.

Implementation

There are basically 5 classes in the code:

  • generator. The generative model in seqGAN that is a one-layer basic LSTM.
  • discriminator. The discriminative model of seqGAN that is CNN.
  • target_lstm. The oracle model that stands for the human distribution.
  • rollout. The roll out policy model.
  • dataloader. The toolkit for data process.

The following subsections introduce some functions.

generator

generator.create_recurrent_unit

This function creates a basic LSTM with three gates (input gate, forget gate and output gate) and hidden states. The input params are used for gradient computation.

generator.create_output_unit

This function is the output unit of the lstm. The input params are also used for gradient computation.

generator.generate

Namely, it generates the [batch_size, sequence_length] data.

generator.pretrain_loss

In pre-training, MLE is used as objective function. The loss function is to minimize it as $-\mathbb{E}_{x \sim p} \log q(x)$.

generator.g_loss

In the training, policy gradient is utilized. The evaluation metric is defined as:
$$
\mathrm{NLL}_{\mathrm{oracle}} = -\mathbb{E}_{Y_1:T \sim G_\theta} [\sum^{T}_{t=1} \log G_{\mathrm{oracle}} (y_t | Y_{1:t-1})]
$$

where $G_\theta$ and $G_{\mathrm{oracle}}$ denote our generative model and the oracle respectively.

discriminator

The discriminator is a CNN for text classification, which uses an embedding layer, followed by a convolutional, max-pooling and softmax layer.

highway

This function implements a highway network (Link). The formula of the highway network is:
$$
t = \mathrm{sigmoid}(Wy+b) \\
z = t \cdot g(Wy + b) + (1 - t) \cdot y
$$

rollout

rollout.get_reward

The RL reward signal comes from the GAN discriminator judged on a complete sequence, and is passed back to the intermediate state-action steps using MCTS.

Gen_Data_loader&Dis_dataloader

Gen_Data_loader.create_batches

This function creates batchs for generator given the file of data.

Dis_dataloader.load_train_data

This function creates batchs by simultaneously given positive file and negative file of data.

Synthetic Data Experiment

The synthetic data experiment was operated on the code. The results are showed in the paper and displayed in the previous article.

To evaluate the efficacy of SeqGAN, the author used the oracle model to represent the real data distribution. The oracle model in this experiment is initialized with random method. Therefore, the objective is to train the generative model closer to the oracle model using policy gradient and GAN.

The flow of training steps is showed:

Preliminaries

Firstly, initialize the oracle model randomly.

After we get the oracle model, we have the model output data set $S$ as ground truth in real data file.

Then, initialize the $G_\theta$, $D_\phi$ randomly.

Pre-training

In the before step, the true data set $S$ has been obtained. Therefore, we train the $G_\theta$ by train set $S$. And MLE is used as loss function.

The $D_\phi$ is pre-trained by minimizing the cross entropy.

Two pre-training processes are both supervised.

Training

Finally, adversarial training takes place. The train set $S$ is used as positive samples and the data outputted by $G_theta$ is used as negative samples.

In every epoch, we first train $G$ by policy gradient in one loop and then train $D$ by the positive and negative samples in several loops.
