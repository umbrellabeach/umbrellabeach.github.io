Linear Regression原理及实现

推导

假设y(label)为实数，且可以用x(features)的线性组合表示，即：

$$
\mathbf{y} = \sum_{i=0}^d w_i x_i
$$

可以得到该hypothesis的表达形式：

$$
h(\mathbf{X})=\mathbf{W^T X}
$$

最小二乘法

最小二乘法可以从Cost/Loss function角度去想，这是统计（机器）学习里面一个重要概念，一般建立模型就是让loss function最小，而最小二乘法可以认为是 loss function = （y_hat -y )^2的一个特例，类似的想各位说的还可以用各种距离度量来作为loss function而不仅仅是欧氏距离。所以loss function可以说是一种更一般化的说法。

引自 知乎-最大似然估计和最小二乘法怎么理解？

对于Regression，通常使用平方错误，则：

$$
err(\hat{y},y)=(\hat{y}-y)^2
$$

Cost Function或者 Loss Function为：

$$
E_{in}(\mathbf{W})=\frac{1}{N} \sum_{i=1}^N (\mathbf{W^T}x_n-y_n)^2
$$
我们所要做的就是让损失达到最小，即求解这样的最优化问题（简化为一个向量的内积）

$$
\min \limits_{\mathbf{W}} E_{in}(\mathbf{w})=\frac{1}{N} \sum_{i=1}^N (\mathbf{W^T}x_n-y_n)^2 = \frac{1}{N} \| \mathbf{XW}-\mathbf{y} \| ^2
$$

通过矩阵的线性代数的计算，梯度为：
$$
\nabla E_{in}(\mathbf{w}) = \frac{2}{N} (\mathbf{X^T X W}-\mathbf{X^T y})
$$

有两种方法最优化，分别为正规方程和梯度下降法。

正规方程

使$\nabla E_{in}(\mathbf{w})=0$，可得以下解：

$$
\mathbf{W_{LIN}}=(\mathbf{X^T X})^{-1} \mathbf{X^T} \mathbf{y}
$$

最小二乘法使用的条件是$\mathbf{X^T X}$的逆矩阵存在，则$\mathbf{X}$要为满秩矩阵。

梯度下降法

梯度下降的算法主要过程：

  1. 计算 $\nabla E_{in}(\mathbf{w_t})$
  2. 更新 $\mathbf{w_{t+1}} \longleftarrow \nabla E_{in}(\mathbf{w_t})$

不断计算上两步，直到$\nabla E_{in}(\mathbf{w_{t+1}})=0$或者足够的迭代次数。

而linear regression的梯度上面已经得出

$$
\nabla E_{in}(\mathbf{w}) = \frac{2}{N} (\mathbf{X^T X W}-\mathbf{X^T y})
$$

在代码实现时候，由于learning rate $\eta $设置为0.1，相对较大，导致梯度下降失败，最后甚至上升到e32！最后$\eta$设为0.0001，迭代1000次之后得到合适的解。

Leetcode 4: Median of Two Sorted Arrays

The problem

There are two sorted arrays nums1 and nums2 of size m and n respectively.

Find the median of the two sorted arrays. The overall run time complexity should be O(log (m+n)).

The thinking

The solution is asked for O(log(m+n)) in overall run time complexity.

At first, I quickly figured out the solution which merges the two arrays in order. It’s easy to code this solution and this solution was acceptable to Leetcode.

Here is the code:

However, this solution takes O(m+n) time complexity and O(m+n) space complexity. We can set two points to move in arrays instead of merging arrays and let points stop at the n/2. This takes O((m+n)/2)=O(m+n) time complexity and O(1) space complexity.

According to the article, this problem can be resolved by a divide-and-conquer strategy. Generally, the problem can be regarded as a special case of finds the k-th element in two arrays.

Given two sorted arrays A and B, we need to find the k-th element in two arrays. From the basic idea of binary search which takes O(logN) time complexity, we can design a divide-and-conquer method. Assume that the length of A and B are both larger than k/2. Then we pick out the A[k/2-1] and B[k/2-1]. There are three cases:

  • A[k/2-1] = B[k/2-1]. We can immediately obtain A[k/2-1] or B[k/2-1] as result. It’s because that A[k/2-1] is the k/2-th elements in A, and so B[k/2-1] is. So A[k/2-1] or B[k/2-1] is the k-th element in A and B.
  • A[k/2-1] < B[k/2-1]. In this case, we can remove the A[0], A[1], …, A[k/2-1] elements in A. Because they can not the k-th element. Then this problem is transferred as find the (k-kp)-th elment in A[pa:]+B[:] or A[:]+B[pb:].
  • A[k/2-1] > B[k/2-1]. It’s the same as above.

We can set A as the smaller array.

Pay attention to this boundary conditions:

  • A or B is empty. Return B[k-1] or A[k-1].
  • k=1. Return min(A[0], B[0]).
  • A[k/2-1]==B[k/2-1]. Return one of A[k/2-1] or B[k/2-1].

PLA算法实现(Python)

本文是学习台大机器学习基石PLA部分的小结。Perceptron Learning Algorithm是最基础的机器学习分类器之一。

PLA的使用条件

如果N维数据为线性可分（Linearly Separable），则可以在N-1维上找一个面将数据无错地分类。

PLA推导

perceptron的假设：

$$
\color{red}{h}(x)= \text{sign}((\sum_{i=1}^d \color{red}{W_i} X_i)-\color{red}{\text{threshold}})
$$
perceptron的简化：

$$
\begin{eqnarray}
\color{red}{h}(x) &=& \text{sign}((\sum_{i=1}^d \color{red}{W_i} X_i)+\color{blue}{(-\text{threshold})(+1)}) \nonumber \\
&=& \text{sign}(\sum_{\color{blue}{i=0}}^d W_i \cdot X_i) \nonumber \\
&=& \text{sign}(\color{red}{\bf{W^T}}\cdot X) \nonumber
\end{eqnarray}
$$

根据上面的简化，在之后的运算过程中，以二维空间为例，数据点$x_i=(a,b)$将会被转化为$x_i=(a,b,1)$。得到的分割线为$w_2 \cdot x_2+w_1 \cdot x_1+b=0$，其中$(w_2,w_1)$为法向量。

“知错能改”原理

PLA算法的“知错能改”原理参见下图

可以知道Y是W与X的内积的sign，而

$$
W\cdot X=|W||X|\cos(W,X)
$$

其值其实是被$\cos(W,X)$影响。
对于$W_t$中一个犯错的点$(X_n(t),Y_n(t))$，如果$y=+1$，那么说明$\cos(W,Y) < 0$，即夹角太大，所以$W_{t+1} = W_t-X_n(t)(+1)$。同理$y=-1$时，$W_{t+1} = W_t-X_n(t)(-1)$。以此达到知错能改的效果。

PLA算法过程

1）找到一个犯错的点$ \text{sign}(\bf{W} \cdot \bf{x_i}) \neq y_i$
2）更新$\bf{W_{\text{new}}} = \bf{W_{\text{old}}} + y_i \cdot \bf{x_i}$
3) 不断循环1、2直到所有点不再犯错

Pocket Algorithm

当样本存在噪声时候，显然不能找到一条可以无错分割的线。如果要寻找犯错误最少的那条线，则：
$$
\bf{w_g} \leftarrow \mathop{argmin}\limits_{w} \sum_{n=1}^N [ y_n \neq \text{sign}(\bf{W}^T \bf{X}_n)]
$$

Pocket算法过程

1）随机找到一个犯错的点$ \text{sign}(\bf{W} \cdot \bf{x_i}) \neq y_i$
2）计算$\bf{W_{\text{new}}} = \bf{W_{\text{old}}} + y_i \cdot \bf{x_i}$
3) 如果$w_{t+1}$在全部数据集上犯的错误比$\hat{w}$要少，则把$\hat{w}$设为$w_{t+1}$
不断循环上述过程直到迭代了提前设置好的次数后，最终$\hat{w}$即为所求$w$

#!/usr/bin/env python
# -*- coding=utf-8 -*-
# Author: fujiaqi@zju.edu.cn

import numpy as np
import random
from display import *

N_ITER = 200


def sign(w, data):
    num = np.dot(w.T, data)
    if num > 0:
        return 1
    return -1


def update_w(w, data, id):
    pred = sign(w, data)
    if pred == id:
        return w
    else:
        w = w + id * data
    return w


def err_w(w, data_set, ids):
    err_num = 0
    for idx in range(0, len(data_set)):
        pred = sign(w, data_set[idx])
        if pred != ids[idx]:
            err_num += 1
    return err_num


def train_w(w, data_set, ids):
    w_opt = w
    err_w_o = err_w(w, data_set, ids)
    for t in range(0, N_ITER):
        print 'iteration = ', t
        # randomly select a sample
        idx = random.randint(0, len(data_set) - 1)
        w = update_w(w, data_set[idx], ids[idx])
        err_w_c = err_w(w, data_set, ids)
        if err_w_c < err_w_o:
            print 'w changed from ', w_opt, ' to ', w
            w_opt = w
            err_w_o = err_w_c

    return w_opt


if __name__ == '__main__':
    data_set = np.array([
        (1, 1, 1), (2, 1, 1), (1, 2, 1),
        (3, 1, 1), (2, 2, 1), (1, 3, 1), (3, 3, 1)
    ])

    ids = [
        1, 1, 1, -1, -1, -1, 1
    ]

    w = np.array([0, 0, 0])
    w = train_w(w, data_set, ids)
    # plot_sample(data_set, ids)
    plot_sample_with_line(data_set, ids, w)
